# BrainOS 桌面端设计系统

本文件由 UI-UX Pro Max 推荐结果与已批准产品流程共同确定，是 BrainOS 前端的视觉和交互基线。

## 1. 设计方向

- 产品类型：企业 AI 知识库后台。
- 风格：Minimalism + Swiss Style。
- 关键词：简约、专业、清晰、克制、易操作。
- 平台：PC Web；验收视口为 1024px 与 1440px，不开发移动端布局。
- 原则：功能优先于装饰；高信息密度区域使用表格和列表，避免堆叠卡片与无意义渐变。

## 2. 色彩

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--color-primary` | `#2563EB` | 主按钮、选中导航、链接、图表主线 |
| `--color-primary-hover` | `#1D4ED8` | 主色悬停 |
| `--color-heading` | `#1E293B` | 标题和高强调文字 |
| `--color-text` | `#334155` | 正文和控件文字 |
| `--color-muted` | `#64748B` | 辅助说明，保证浅色背景可读性 |
| `--color-background` | `#F8FAFC` | 应用主背景 |
| `--color-surface` | `#FFFFFF` | 面板、表格和弹窗 |
| `--color-border` | `#E2E8F0` | 分隔线与默认边框 |
| `--color-success` | `#16A34A` | 成功、可用 |
| `--color-warning` | `#D97706` | 处理中、提醒 |
| `--color-danger` | `#DC2626` | 失败、删除 |

不使用大面积渐变、霓虹色、玻璃拟态或低对比度灰字。状态必须同时显示文字或图标，不只依赖颜色。

## 3. 字体

- 拉丁与数字：Inter。
- 中文：PingFang SC、Microsoft YaHei、sans-serif。
- 页面标题：24px / 32px，600。
- 区块标题：18px / 28px，600。
- 正文：14px / 22px，400。
- 控件文字：14px / 20px，500。
- 辅助文字：12px / 18px，400。
- 数据指标：28px / 36px，600，使用等宽数字特性。

所有按钮、输入框、标签、表格和侧栏项显式定义字号与行高，不依赖浏览器默认值。

## 4. 布局

- 左侧导航宽 224px，折叠宽 64px；顶部栏高 64px。
- 主内容区最大宽度不强制居中，使用 24px 页面内边距；1024px 视口使用 20px。
- 页面垂直间距以 8px 为基准，常用 8、12、16、24、32。
- 圆角只使用 6px、8px、12px；业务表格和大面板不使用夸张圆角。
- 阴影仅用于弹窗、下拉层和悬浮上传区域；常规面板以边框和留白分层。
- 页面不嵌套多层卡片，不用 bento grid 代替表格或列表。

## 5. 导航与页面

- 左侧导航顺序：工作台、知识库、AI 问答；管理员额外显示用户管理、操作日志。
- 文档管理位于知识库详情内，不增加一级导航。
- 顶部栏显示页面标题、必要的返回/面包屑、用户菜单；不堆放状态组件。
- 当前导航使用主色背景弱强调、主色图标和 600 字重。
- 登录页只包含品牌名、标题、用户名、密码、提交按钮和错误提示。

## 6. 组件规则

### 按钮

- 每个区域最多一个主按钮；次要操作使用默认或文字按钮。
- 危险操作使用红色文字或危险按钮，并要求二次确认。
- 悬停只改变颜色、边框或阴影，不使用缩放造成布局跳动。
- 交互过渡为 150-250ms；所有可点击元素显示指针光标。

### 表单

- 标签始终可见，不使用占位符替代字段名。
- 校验错误显示在对应字段下方，并说明修复方法。
- 新建/编辑使用宽度适中的对话框；复杂上传流程使用页面内区域。
- 提交期间禁用重复提交并显示加载状态。

### 表格

- 表格用于文档、用户和日志；固定操作列，长文本省略并可查看完整内容。
- 提供加载骨架、空状态、错误状态和分页。
- 批量操作只在确有需求的文档列表中出现，不为完成度而增加控制项。

### 图标

- 使用 Element Plus Icons 或统一的 Lucide 线性图标，禁止使用 emoji 作为界面图标。
- 默认 18px，按钮内 16px，空状态 40px；保持统一描边风格。
- 箭头、展开和分页使用 SVG 图标，不使用文本符号替代。

## 7. 页面特定规则

### 工作台

- 首行四张等宽指标卡；只显示名称、真实数值和简短辅助变化信息。
- 第二行左侧为七日问答折线图，右侧为最近文档表格。
- 图表使用单条 `#2563EB` 折线、轻量网格和明确 Tooltip，不增加饼图或装饰性仪表盘。

### 知识库

- 使用最多三列的轻量卡片列表；卡片包含名称、描述、文档数、更新时间和进入操作。
- 新建入口置于页面右上角；空状态直接提供新建按钮。
- 卡片悬停只提升边框和轻微阴影，不位移或缩放。

### 文档管理

- 顶部为知识库摘要和上传区域，下方为文档表格。
- 处理状态同时使用小图标、标签色和文字。
- 失败原因可直接查看；重试与删除靠近状态列，避免隐藏在多级菜单。

### AI 问答

- 左侧会话栏宽 260px；中间对话区保持舒适阅读宽度。
- 顶部仅保留知识库选择和模型选择；发送按钮是唯一主操作。
- 用户消息与 AI 消息视觉区分但不过度气泡化；AI 正文支持 Markdown。
- 引用紧跟回答，以“来源 1、来源 2”折叠项呈现文件名、页码和命中片段。
- 流式阶段显示停止按钮；失败状态保留问题并提供重试。

### 用户与日志

- 使用高可扫读表格，不转换为卡片网格。
- 筛选项保持单行并提供重置；筛选结果为空时给出调整条件建议。

## 8. 可访问性与反馈

- 正文与背景对比度至少 4.5:1；键盘焦点轮廓始终可见。
- 使用语义化 `nav`、`main`、`header`、`button`、`table` 和标题层级。
- Tab 顺序与视觉顺序一致，不设置键盘陷阱。
- 页面级加载、局部加载、空状态、成功和错误状态互相区分。
- 动画遵循 `prefers-reduced-motion`；不使用自动播放媒体。
- 1024px 与 1440px 视口均不得出现主要内容遮挡或整页横向滚动。

## 9. 禁止项

- 不使用 emoji 图标、装饰性徽章、营销式 Hero、霓虹渐变或玻璃拟态。
- 不把所有内容放入卡片，不使用巨型圆角容器包裹整页。
- 不制作暗黑模式、移动端菜单、复杂主题切换或可视化大屏。
- 不添加未经 Spec 记录的页面、导航项、统计图或批量功能。
