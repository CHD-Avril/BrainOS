# BrainOS 答辩演示脚本

建议时长 7–10 分钟。演示前确保 MySQL、Redis、Chroma、后端和前端已启动，`QWEN_API_KEY` 可用。

## 0. 演示前准备

- 登录账号：`admin`
- 开发演示密码：`BrainOS@123`
- 演示文档：`fixtures/employee-handbook.md`
- 如数据库中已有同名知识库，可改用“员工制度演示”。

## 1. 登录与工作台（1 分钟）

1. 输入管理员账号并登录。
2. 介绍左侧 5 个入口：工作台、知识库、AI 问答、用户管理、操作日志。
3. 指出工作台中 4 项指标来自真实数据库，7 天趋势会补齐无提问的日期。

预期：页面无占位数据，空库显示 0，有数据时指标实时变化。

## 2. 创建知识库与导入（2 分钟）

1. 进入“知识库”，点击“新建知识库”。
2. 名称填“员工制度”，描述填“人事与行政制度演示”。
3. 进入文档管理，上传 `fixtures/employee-handbook.md`。
4. 观察状态：“解析中” → “索引中” → “可用”。

讲解：后端先校验文件和 SHA-256 去重，然后用 Tika 解析、切片，调用千问 Embedding，最后写入 Chroma。

预期：文档显示“可用”和大于 0 的切片数。

## 3. RAG 问答与引用（3 分钟）

1. 进入“AI 问答”，选择“员工制度”和“通义千问”。
2. 提问：`正式员工每年有几天年假，如何申请？`
3. 展示流式输出和可展开的来源。
4. 展开引用，指出文件名、片段摘要和相似度。
5. 再问一个文档没有的问题：`公司今年的股票价格是多少？`

预期：第一问回答“5 天”且附引用；第二问返回“当前知识库中未找到可靠依据”，不编造答案。

可选：新建会话并切换 DeepSeek，说明模型路由与知识检索相互独立。

## 4. 企业管理功能（2 分钟）

1. 进入“用户管理”，新建普通用户 `demo.user`。
2. 展示角色、状态、最近登录和启停操作。
3. 说明系统会拒绝停用或降级最后一名可用管理员。
4. 进入“操作日志”，选择“创建用户”过滤。

预期：能看到刚才的用户创建和文档上传记录，日志不包含密码、Token 或 API Key。

## 5. 收尾（1 分钟）

总结三点：

1. 不是静态页面：文档真实解析、向量化并进入 Chroma。
2. 回答可追溯：检索限定知识库，有阈值、固定兜底和文档引用。
3. 具备企业后台完整性：权限、令牌撤销、用户管理、审计和自动化测试齐全。

## 现场应急

- 模型 API 短暂不可用：优先演示文档已入库、历史会话、工作台和审计，并说明模型错误会被安全转换。
- 文档未及时变为可用：先检查 Chroma 健康和千问 Key，失败时点击“重试”。
- 现场网络不稳定：提前保留一个已索引知识库和已生成会话，不要在答辩前清空 Docker 卷。
